# ChatDot 手动发版冒烟测试清单 (Manual Test Checklist)

根据插件的业务逻辑和最近修改，以下场景需要在提交 Chrome Web Store 前，由人类进行真机本地冒烟测试：

## 一、前期准备
- [ ] 确保解压了 `release.zip` 并通过 Chrome 的 "加载已解压的扩展程序" (Load unpacked) 进行加载。
- [ ] 打开或刷新一个 ChatGPT (https://chatgpt.com) 对话页面，确认包含一定数量的用户与 AI 问答环节。

## 二、功能正确性
- [ ] **侧边栏加载测试**：观察是否在对话界面成功渲染了右下角的导航侧边栏（上方通常是一个大纲按钮，下方为四向导航键）。
- [ ] **大纲面板渲染**：
  - 点击顶部的“大纲”按钮，是否正常展开面板。
  - 面板列表是否提取到了对话中的用户消息。
  - 点击外部空白区域（未钉住时），大纲是否可以正确隐藏。
- [ ] **导航与悬浮预览测试**：
  - 鼠标悬停在“上一条”、“下一条”按钮上方，是否正确展示浮窗说明（含部分提取的当前内容）。
  - 点击各个方向按键时，视图是否能正确定位到目标 DOM 和顶部/底部。
  - 观察计数器 `X/Y` 目前是否正确计算展示了当前滚动条所处在的用户消息进度。

## 三、Popup 与 设置项联动
- [ ] **启用与停用**：在右上角点击扩展图标打开 Popup 面板，切换“启用导航”开关，在 ChatGPT 页面查看侧边栏是否立刻响应出现/隐藏。
- [ ] **多视图开关响应**：
  - 切换“悬浮预览”开关，回到页面测试悬浮是否被禁止。
  - 切换“大纲”开关，页面右下角侧边栏的第一按钮是否动态出现/消失。
  - 切换滚动模式（平滑/瞬移），页面导航是否跟随模式改变跳跃方案。
- [ ] **I18n 兼容性**：点击 Popup 面板右上角语言图标，切换为 `English/日本語` 等语言视图，观察是否秒切换展示文字。

## 四、健壮性测试
- [ ] **SPA 跳转兼容（难点）**：从 ChatGPT 会话 A 跳转到会话 B（点击左侧历史记录列表），观察旧侧边栏是否被卸载，且新侧边栏是否重新就绪正确读取会话 B 的长度与数据。
- [ ] **缩放拉伸**：改变浏览器窗口大小使其产生重新排版布局后，导航侧边栏是否还能维持在右下角不被遮挡。
